Super ! Tes conteneurs sont enfin tous "Started" et "Running". Tu as maintenant une infrastructure complète avec PHP, Python, MariaDB et Nginx.
Voici ton "kit de survie" des commandes Docker les plus utiles pour gérer ton projet SFE au quotidien.
_____________________________________________________________________________________________________________________________________________________________
1. Gestion des conteneurs (Docker Compose)
À lancer depuis le dossier sfe-docker-nginx sur ton Windows :
•	Lancer le projet : docker-compose up -d
•	Arrêter le projet : docker-compose stop
•	Tout éteindre et supprimer les réseaux : docker-compose down
•	Reconstruire après une modif du Dockerfile : docker-compose up --build -d
•	Voir l'état de tes services : docker-compose ps
______________________________________________________________________________________________________________________________________________
2. Entrer dans les terminaux (Le plus utile pour toi)
Si tu veux taper des commandes à l'intérieur de tes machines :
•	Entrer dans Python : docker exec -it python_service bash
•	Entrer dans PHP : docker exec -it php_fpm bash
•	Entrer dans la base de données : docker exec -it mariadb_db mariadb -u root -p
_____________________________________________________________________________________________________________________________________________
3. Exécuter tes scripts directement
Tu n'es pas obligé d'entrer dans le conteneur, tu peux lancer une commande depuis Windows :
•	Lancer ta prédiction Python :
docker exec -it python_service python3 cross_path_arbre.py
•	Vérifier les logs si un script plante :
docker-compose logs -f python (le -f permet de voir les erreurs en temps réel).
______________________________________________________________________________________________________________________________________________________
4. Nettoyage (Si tu manques de place)
•	Supprimer les images inutilisées : docker image prune
•	Tout nettoyer (volumes compris - ATTENTION) : docker system prune -a --volumes
_______________________________________________________________________________________________________________________________________________________
💡 Petit conseil pour ton projet
Maintenant que ton service Python est séparé du service PHP, si tu veux que ton site Web appelle ton script Python, tu as deux options :
1.	Soit PHP lance une commande exec() via Docker (un peu complexe).
2.	Soit (le plus simple) tu continues d'installer Python sur le conteneur PHP comme on le faisait au début, et tu gardes le conteneur Python juste pour tes tests et tes générations d'arbres complexes.
